[gitの初期設定] (作業ディレクトリ内) git init [ステージ上に加える] git add [状態を確認する] git status [コミットをする] git commit [差分を確認する] git diff [変更履歴を確認する] git log [ファイルの削除を記録する]　 # ファイルごと削除をする git rm <ファイル名> [ファイルの移動を記録する]' git mv <旧ファイル><新ファイル> ローカルの内容をリモートリポジトリに送ること = push [リモートリポジトリを新規追加する] git remote add origin <リモートリポジトリのurl> [リモートリポジトリへ送信する] git push <リモート名> <ブランチ名> [ファイルへの変更を取り消す] git checkout -- <ファイル名> [ステージした変更を取り消す](ワークツリーには影響を与えない) git reset HEAD [直前のコミットをやり直す](リモートにpush済のものには適用できない) git commit --amend [リモートの状態を確認する] git remote # URLを表示 git remote -v [リモートリポジトリを追加] git remote add <リモート名><リモートURL> [リモートから取得] 1. git fetch <リモート名> (ワークツリーには反映されない) 2. git pull <リモート名><ブランチ名> (ワークツリーにまで反映する) [リモートの詳細情報を表示する] git remote show <リモート名> [リモートの変更・削除] (変更) git remote rename <旧リモート名> <新リモート名> (削除) git remote rm <リモート名> [ブランチを作成] git branch <ブランチ名> (切り替えは行わない) [ブランチ一覧を確認する] git branch [ブランチを切り替える] git checkout <ブランチ名> [変更履歴をマージする] git merge <ブランチ名> git merge <リモート名/ブランチ名> [conflict]
